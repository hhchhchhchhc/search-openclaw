# FAQ

## OpenClaw 的 Web Search 为什么这么关键？

因为对 Agent 来说，搜索不是附属能力，而是基础能力。

它决定了几件核心的事：

- 能不能拿到最新信息
- 能不能做多来源交叉验证
- 能不能稳定找到高质量延伸阅读
- 能不能在复杂任务里持续补充上下文

很多时候大家觉得“模型不够聪明”，实际是搜索层没有搭好。

## 第一次配置，应该先选哪个？

最简单的经验判断：

- 有卡：优先 `Brave`
- 没卡：优先 `Tavily`

这是最容易少踩坑的起步方式。

## 完全免费路线够不够用？

对个人测试、轻度使用、体验 OpenClaw 搜索能力来说，通常够用。

但如果你要长期跑高频任务，或者要把它作为稳定生产能力，还是要考虑：

- 免费额度可能变化
- 接口策略可能变化
- 单一免费源可能不够稳

所以免费路线适合起步，不一定适合长期唯一依赖。

## 什么时候该用多搜索源组合？

当你遇到下面这些情况时，就值得上多搜索源：

- 单一搜索源经常漏掉关键结果
- 不同任务对搜索源偏好差异明显
- 你需要更强的容灾能力
- 你在做研究、持续监控、热点跟踪

多搜索源的核心价值，不是“更酷”，而是降低单点故障风险。

## Brave 和 Tavily 怎么选？

可以直接按下面理解：

- `Brave`：更适合有卡、想少折腾、重视兼容性的人
- `Tavily`：更适合不想绑卡、想快速跑通链路的人

如果你懒得选，就先从这两个里挑一个开始，不必一开始就上复杂组合。

## Exa / You.com / iFlow 适合谁？

更适合已经把主搜索跑通的人。

这些路线的价值通常在于：

- 继续压低成本
- 增强语义搜索能力
- 提高组合灵活性
- 做多源补盲

其中 `iFlow` 在这个仓库里还有一个额外优势：如果你当前电脑上的 OpenClaw 已经配置了 iFlow，`search-openclaw` 会直接复用那份 key 和 base URL，不用再维护第二份。

它们未必是最省事的起点，但对进阶用户有意义。

## 这个仓库后面会放什么？

- 不同搜索路线的适用场景
- 免费额度和计费变化

- 实战踩坑记录
