# 搜索路线

## 免费优先

给 OpenClaw 配搜索，不是比“谁更全”，而是比“谁更适合 Agent”。

下面这几条路线，各自适合不同约束。

## 1. Brave

社区常提每月约 `2000` 次免费，速度快，结构化结果好，但通常要绑卡。

适合：

- 有卡
- 想少折腾
- 重视稳定性
- 想优先走主流配置

## 2. Tavily

社区常提每月约 `1000` 次免费，不绑卡，对 Agent 友好，开箱即用。

适合：

- 刚入门
- 不想绑卡
- 想先快速验证搜索链路
- 希望低门槛开始

## 3. Exa / You.com

更偏语义搜索和赠送额度思路，适合继续压低成本。

适合：

- 已经有主力搜索源
- 想补充语义检索能力
- 想把成本继续往下打
- 愿意自己做更多测试

## 4. iFlow

如果你本机已经在用 OpenClaw，而且 OpenClaw 默认模型就是 `iflow`，那么这条路线的特点是：

- 不需要再单独维护一份 key
- 可以直接复用 `~/.openclaw/openclaw.json` 里的 iFlow 配置
- 适合把搜索层和当前 OpenClaw 运行环境对齐

这条路线在 `search-openclaw` 里更像研究摘要增强，不是传统搜索 API。

## 5. 多搜索源组合

适合把多个搜索源拼起来，做容灾和补盲。

单一搜索源最大的问题，不是“平时不好用”，而是：

- 某些时候搜不到
- 某些时候结果偏科
- 某些时候接口抽风

多引擎方案的意义，就是让 OpenClaw 在不同来源之间切换和互补，减少单点故障。

## 推荐组合

### 方案 A：最省心

- 主搜索：`Brave`
- 备用：`Tavily`

### 方案 B：完全免费优先

- 主搜索：`Tavily`
- 补充：`Exa` 或 `You.com`

### 方案 C：稳定性优先

- 主搜索：`Brave`
- 备用：`Tavily`
- 容灾：多搜索源组合

### 方案 D：进阶折腾流

- 主搜索：`Brave` 或 `Tavily`
- 研究增强：`iFlow`
- 多源补盲：多搜索源组合
